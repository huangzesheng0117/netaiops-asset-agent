# V2 Batch47 上下文跟随执行、批量确认执行与汇总分析

## 目标

Batch47 修复用户在前端说“将你上述给出的命令执行一下并分析”时，系统无法理解上下文引用的问题。

## 新增能力

1. 识别上下文执行意图：
   - 将你上述给出的命令在设备上执行
   - 执行你刚才给的命令
   - 把上面的命令执行一下
   - 根据命令结果给出分析

2. 不带 YES 时不执行，只提示确认：
   - 确认执行全部命令 YES

3. 支持批量执行上一轮所有 passed 命令。

4. 批量执行后输出：
   - 执行统计
   - 命令执行结果摘要
   - 综合分析
   - 分命令关键分析
   - 建议下一步

## 安全边界

1. 只有上一轮 pending commands 中 guard_status=passed 的命令会被执行。
2. review 和 blocked 命令不会被批量执行。
3. 必须显式包含 YES。
4. 批量执行数量默认上限为 8。
5. 每条命令仍通过 ConfirmedNetmikoExecutor 再次校验和审计。
6. 不暴露 Netmiko MCP 配置工具。

## 前端测试方式

第一轮输入：

    WG88-SW-H15-1当前CPU利用率异常，给我第一批排查命令

第二轮输入：

    将你上述给出的命令在设备上执行，然后根据命令的结果给出分析

预期：不执行，提示需要确认。

第三轮输入：

    确认执行全部命令 YES

预期：执行上一轮所有 passed 只读命令，并输出汇总分析。
