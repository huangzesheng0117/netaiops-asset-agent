# NetAIOps 网络信息统一查询入口 V1 安全说明

## 1. 只读边界

V1 只调用 CMDB 网络设备查询接口，不提供设备配置修改、CMDB 写入、删除、变更等能力。

后端 Tool 层当前主要包含：

- query_cmdb_devices
- query_cmdb_device_detail
- query_cmdb_devices_by_ips

均为只读查询。

## 2. LLM 使用边界

公司本地 LLM 只负责解析用户意图，生成结构化 Tool Plan。真实数据仍由后端 CMDB Tool 查询，LLM 不直接访问 CMDB Token，不直接拼接 SQL，不直接访问数据库。

处理链路：

    用户自然语言
      -> 公司本地 LLM 解析为 Tool Plan
      -> 后端白名单校验字段和过滤条件
      -> 后端 CMDB Tool 只读查询
      -> 返回真实 CMDB 数据

## 3. Token 与 Key 管理

敏感配置位于：

- /etc/netaiops-asset-agent/asset-agent.env
- /etc/netaiops-asset-agent/config.yaml

文件权限应保持为 640 或更严格。

前端和 /api/v1/llm/config 只返回 api_key_configured=true/false，不返回真实 API Key。

## 4. 审计与日志

平台会记录查询审计信息，包括问题、工具名、查询条件、返回条数等。审计日志不应记录真实 API Key。

## 5. 当前暂缓项

访问控制暂缓到后续版本统一建设。正式推广前建议接入 Nginx Basic Auth 或公司统一认证入口。
